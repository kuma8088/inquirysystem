# 受託開発事業の計画テンプレート

クライアントワーク・受託開発事業特有の計画・KPI・マイルストーン。

## 受託開発固有の KPI

| 指標 | 説明 | 目標設定例 |
|:---|:---|:---|
| **月間売上** | 総収入 | ¥30万 → ¥60万 → ¥100万 |
| **単価** | 月単価 or 時給 | ¥50万/月 → ¥80万/月 |
| **稼働率** | 案件稼働日数/営業日 | 60% → 80% → 90% |
| **直請け比率** | エージェント非経由 | 0% → 30% → 50% |
| **リピート率** | 継続・再発注 | > 50% |

## エージェント vs 直請け

| 項目 | エージェント | 直請け |
|:---|:---|:---|
| 手数料 | 10-30% | 0% |
| 営業負荷 | 低 | 高 |
| 単価交渉 | エージェント経由 | 直接 |
| 関係構築 | 間接的 | 直接的 |
| 契約の安定性 | 中（切られやすい） | 高（関係次第） |

### 直請け獲得の方法

```
1. 既存人脈からの紹介
2. 勉強会・イベントでの出会い
3. SNS/ブログ経由の問い合わせ
4. 過去クライアントからの再発注
5. 知人の会社への直接提案
```

## リスク管理

| リスク | 対策 |
|:---|:---|
| 案件が見つからない | 常に 3件並行で探す |
| 契約終了 | 終了 2ヶ月前から次を探す |
| 単価が上がらない | スキルアップ + 実績アピール |
| 炎上案件 | 契約前に見極め、逃げる勇気 |
| スキル陳腐化 | 案件の 10% を学習に充てる |

## 稼働と学習のバランス

| フェーズ | 稼働 | 学習 | 営業 |
|:---|:---:|:---:|:---:|
| 案件探し中 | 0% | 50% | 50% |
| 案件稼働中 | 80% | 10% | 10% |
| 案件終了前 | 70% | 10% | 20% |

# コンテンツ系ポートフォリオ

技術発信・記事・登壇など、GitHub 以外のポートフォリオの追加観点。

## 追加調査項目

Web検索で以下を追加調査：
- 「技術ブログ 人気 記事」
- 「エンジニア 登壇 始め方」
- 「Zenn トレンド」

## コンテンツ系の評価軸

| 軸 | 観点 | 重要度 |
|:---|:---|:---:|
| **独自性** | 他にない視点・切り口か | ⚠️ 高 |
| **実践性** | 実体験に基づいているか | ⚠️ 高 |
| **継続性** | 定期的に更新できるか | ⚠️ 高 |
| **SEO/発見性** | 検索で見つかるか | 中 |
| **エンゲージメント** | 反応を得られているか | 中 |

## コンテンツタイプ

### タイプ1: 技術ブログ記事

**プラットフォーム:**
- Zenn（日本エンジニア向け、収益化可）
- Qiita（日本、認知度高い）
- note（非エンジニアにもリーチ）
- 個人ブログ（ブランディング）

**推奨テーマ（Naoya のスキル活用）:**
| テーマ | 想定読者 | 差別化ポイント |
|:---|:---|:---|
| Terraform ベストプラクティス | インフラエンジニア | 実案件での学び |
| AWS コスト最適化 | 全エンジニア | 具体的な削減額 |
| LPIC 取得体験記 | 資格取得者 | 勉強法、実務での活用 |
| ネットワーク基礎 | 初学者 | 営業経験からの説明力 |

### タイプ2: 登壇・LT

**場所:**
- 社内勉強会（ハードル低）
- connpass イベント
- AWS ユーザーグループ
- SRE/DevOps 勉強会

**推奨トピック:**
- 「Terraform で学んだ N 個の教訓」
- 「AWS 初心者がハマるポイントと解決策」
- 「ネットワークエンジニアが見た AWS VPC」

### タイプ3: OSS コントリビューション

**始めやすいもの:**
- ドキュメント修正（typo、翻訳）
- Issue の再現確認
- 小さな機能追加

**Naoya に向いている OSS:**
- Terraform Provider（AWS）
- AWS CDK
- 日本語ドキュメント翻訳


## 記事テンプレート

```markdown
# [キャッチーなタイトル]

## TL;DR
[3行で要約]

## 背景
[なぜこの記事を書いたか]

## 本題
[技術的な内容]

## 学んだこと
[実践から得た教訓]

## まとめ
[結論と次のアクション]

## 参考
[リンク]
```

```markdown
[具体的な方法]

[実務での活用]
```


## 差別化ポイント

❌ 避けるべき:
- 公式ドキュメントのコピー
- 「〇〇 やってみた」だけ
- 結論のない記事

✅ 目指すべき:
- 実体験に基づく学び
- 具体的な数字・結果
- 独自の視点・切り口
- 次のアクションを示す

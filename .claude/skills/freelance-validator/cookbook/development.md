# 受託開発事業の追加評価軸

クライアントワーク・受託開発系アイデアを評価する際の追加観点。

## 追加調査項目

Web検索で以下を追加調査：
- 「[技術] フリーランス 単価」
- 「[技術] 案件 相場」
- 「[分野] エンジニア 需要」

## 受託開発固有の評価軸

| 軸 | 観点 | 重要度 |
|:---|:---|:---:|
| **案件供給** | 仕事が途切れないか | ⚠️ 高 |
| **単価水準** | 市場相場はいくらか | ⚠️ 高 |
| **参入障壁** | 経験年数・資格要件 | ⚠️ 高 |
| **リモート可否** | 場所を選ばず働けるか | 中 |
| **スキル陳腐化** | 技術が廃れないか | 中 |

## 危険信号 🚨

以下に該当する場合は **Feasibility を1段階下げる**：

- [ ] 経験年数が市場要件未満
- [ ] 単価が相場の50%以下
- [ ] エージェント依存度100%
- [ ] 技術が5年以内に陳腐化リスク
- [ ] 常駐必須で場所の自由度ゼロ

## 成功パターン

1. **技術スタック特化**: 「○○専門」で指名を取る
2. **業界特化**: 金融×AWS、医療×セキュリティ等
3. **直請け比率UP**: エージェント手数料を削減
4. **パッケージ化**: 「○○構築パック ¥XX万」

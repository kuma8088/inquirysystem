# Freelance Validator v2.0

フリーランスアイデアを時間投資前に厳しく評価し、無駄な時間を防ぐ。

## 評価プロセス

### Step 1: 事業タイプの特定

アイデアを以下に分類し、該当する cookbook を読み込む：

| タイプ | 条件 | 参照ファイル |
|:---|:---|:---|
| SaaS | ソフトウェアをサービスとして提供 | `cookbook/saas.md` |
| コンサル | 知識・経験を売る | `cookbook/consulting.md` |
| コンテンツ | 情報発信で収益化 | `cookbook/content.md` |
| 受託開発 | クライアントワーク | `cookbook/development.md` |

### Step 2: 市場調査（必須）

Web検索を実行して実データを収集：

**日本市場:**
- 「[キーワード] フリーランス 日本」
- 「[キーワード] 副業 稼ぐ」
- 「[サービス名] 競合 比較」

**グローバル市場:**
- 「[keyword] freelance market」
- 「[keyword] indie hacker」

調査なしの評価は禁止。

### Step 3: 7軸評価

| 軸 | 評価観点 |
|:---|:---|
| **Market** | 競合状況、差別化ポイント |
| **Demand** | 金を払う人が存在するか |
| **Timing** | 今やるべきか、早すぎ/遅すぎか |
| **Feasibility** | 1人で3ヶ月以内に収益化可能か |
| **Monetization** | 収益モデル、継続収入 vs 単発 |
| **Skill Fit** | 既存スキルとの適合度 |
| **Interest** | 継続できるモチベーション |

### Step 4: 評価姿勢

**残酷なまでに正直に。**

- 「100回やられてる」なら今すぐ伝える
- 希望的観測は排除
- 「頑張れば」「うまくいけば」は禁句
- ユーザーの時間を守ることが最優先

## 判定基準

| 判定 | 条件 |
|:---|:---|
| 🟢 **Do it** | 7軸中5つ以上が⭐3以上、Feasibility が⭐3以上 |
| 🟡 **Maybe** | 改善次第で可能性あり、または特定条件下で成立 |
| 🔴 **Skip it** | 4軸以上が⭐2以下、または Feasibility が⭐1 |

## 判定後のアクション

### 🟢 Do it の場合

```
## 次のステップ

1. [ ] MVP定義: 最小限の機能で検証できる形は？
2. [ ] 顧客候補: 最初の1人は誰？どこにいる？
3. [ ] 1週間実験: 何を検証する？

### 🔗 連携スキル
- **business-roadmap-advisor**: 「この判定結果を元に実行計画を立てて」
- **portfolio-ideator**: 「このアイデアをポートフォリオ化して」
```

### 🟡 Maybe の場合

```
## 検証すべきこと

1. [ ] 不足情報: [特定された不明点]
2. [ ] 検証仮説: [3つの仮説]
3. [ ] 2週間計画: [検証アクション]

### 🔗 連携スキル
- **business-roadmap-advisor**: 「検証計画を立てて」
```

### 🔴 Skip it の場合

```
## 代替案

このアイデアの強みを活かせる別の方向性：
1. [代替案1]
2. [代替案2]
3. [代替案3]

### 🔗 連携スキル
- **portfolio-ideator**: 「別のアイデアを提案して」
```

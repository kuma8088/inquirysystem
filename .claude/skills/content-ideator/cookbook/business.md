# ビジネス層向けコンテンツ戦略

非エンジニア・意思決定者をターゲットにしたコンテンツの追加観点。

## ビジネス層向けの特徴

| 要素 | 重視すべき点 |
|:---|:---|
| **わかりやすさ** | 専門用語を避ける/説明する |
| **ビジネス価値** | ROI、コスト削減、効率化 |
| **具体的な数字** | 「30%削減」「2倍高速化」 |
| **意思決定の材料** | 比較、判断基準 |

## 推奨プラットフォーム

| Platform | 優先度 | 理由 |
|:---|:---:|:---|
| **note** | ⭐⭐⭐⭐⭐ | ビジネス層が多い |
| **LinkedIn** | ⭐⭐⭐⭐ | 意思決定者にリーチ |
| **Twitter/X** | ⭐⭐⭐ | 幅広い層、拡散力 |
| **YouTube** | ⭐⭐⭐ | 視覚的に説明 |
| **Zenn** | ⭐⭐ | 技術寄りだが一部向け |

## コンテンツタイプ

### 1. ビジネス価値訴求

```markdown
# [技術] 導入で [成果] を実現した話

## 結論
[具体的な成果: コスト30%削減、工数半減など]

## 背景
- 課題: [ビジネス課題]
- 制約: [予算、期間、人員]

## 解決策
[技術的な説明を最小限に、何をしたか]

## 成果
| 指標 | Before | After | 改善率 |
|:---|:---:|:---:|:---:|
| コスト | ¥X万 | ¥X万 | -X% |
| 工数 | X時間 | X時間 | -X% |

## 学び
[意思決定者が参考にできるポイント]
```

## 翻訳のコツ

### 技術用語 → ビジネス用語

| 技術用語 | ビジネス表現 |
|:---|:---|
| Infrastructure as Code | インフラ構築の自動化 |
| CI/CD | 開発〜リリースの自動化 |
| スケーラビリティ | 負荷増加への対応力 |
| 可用性 | システムが止まらない仕組み |
| セキュリティ | 情報漏洩・攻撃への対策 |

### 数字で語る

```
❌ 「パフォーマンスが向上しました」
✅ 「処理時間が 5秒 → 0.5秒 に短縮（90%改善）」

❌ 「コスト削減できました」
✅ 「月額 ¥50万 → ¥15万 に削減（年間 ¥420万節約）」
```

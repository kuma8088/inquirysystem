# コンテンツ再利用戦略（Repurpose）

1つのプロジェクト・学びを複数のコンテンツに展開する戦略。

## 再利用の基本原則

```
1つのインプット → 複数のアウトプット
```

| 工数 | 単発 | 再利用 |
|:---|:---:|:---:|
| 1プロジェクト | 1コンテンツ | 5-10コンテンツ |
| 10h 作業 | 4h 執筆 | 4h + 2h + 1h + 1h + 1h |
| ROI | 低 | 高 |

## 再利用パターン

### パターン1: フォーマット変換

```
Zenn 技術記事（2000字）
    ↓
├── Twitter スレッド（10ツイート）
├── YouTube 解説動画（10分）
├── note 裏話（1500字）
└── LinkedIn ビジネス視点（500字）
```

### パターン2: 深度変換

```
詳細チュートリアル（5000字）
    ↓
├── 要約版（1000字）
├── チートシート（1枚）
├── クイックスタート（500字）
└── FAQ 形式（Q&A 10個）
```

### パターン3: ターゲット変換

```
エンジニア向け記事
    ↓
├── 初心者向け入門版
├── 経験者向け応用版
└── ビジネス層向け ROI 版
```

### パターン4: 時間軸変換

```
学習中のメモ
    ↓
├── 学習ログ（進行中）
├── 完了報告（達成時）
├── 振り返り（1ヶ月後）
└── 体系化（3ヶ月後）
```

## 具体例: 1プロジェクトの展開

### 例: Terraform で AWS 環境を構築

```
【プロジェクト完了】
    ↓
Week 1:
├── Zenn: 「Terraform で AWS 3層アーキテクチャを構築する」
└── Twitter: 完成報告 + アーキテクチャ図

Week 2:
├── Twitter スレッド: 「Terraform でハマった5つのポイント」
└── GitHub: コード公開 + README 充実

Week 3:
├── note: 「なぜ Terraform を選んだか」
└── YouTube: ハンズオン動画（検討中）

Week 4:
├── Zenn: 「Terraform ベストプラクティス 10選」
└── LinkedIn: ビジネス価値の視点で要約
```

```markdown
3/5 ② [ポイント2]
[要約 200字]

```

### 技術記事 → note

```markdown
# 技術記事（Zenn）
「Terraform で AWS 環境を構築する方法」
[手順、コード中心]

# note 版
「AWS 構築に Terraform を選んだ理由と学び」
[意思決定プロセス、失敗談、感情を含む]
```

```markdown
1. コスト30%削減
2. 構築時間 1週間 → 1時間
3. ヒューマンエラー 0件

詳細はブログで→ [URL]

#AWS #Terraform #インフラ
```

## 再利用カレンダー

### 1つのプロジェクトを4週間で展開

| Week | Platform | Content Type | 工数 |
|:---:|:---|:---|:---:|
| 1 | Zenn | 詳細チュートリアル | 4h |
| 1 | Twitter | 完成報告 | 15m |
| 2 | Twitter | スレッド化 | 1h |
| 2 | GitHub | コード公開 | 1h |
| 3 | note | 裏話・学び | 2h |
| 3 | Twitter | note 宣伝 | 15m |
| 4 | Zenn | 派生記事 | 3h |
| 4 | LinkedIn | ビジネス視点 | 30m |

**合計: 約12h で 8コンテンツ**

## 再利用チェックリスト

コンテンツ作成後に確認：

```
□ Twitter でシェアしたか？
□ スレッド化できるか？
□ 別ターゲット向けに書き換えられるか？
□ 動画化の価値があるか？
□ 派生記事のネタになるか？
□ 3ヶ月後に振り返り記事を書けるか？
```

## 効率化のコツ

### 1. 最初から再利用を意識

```
記事執筆時に:
- 図解を作っておく（SNS で使える）
- 箇条書きセクションを入れる（スレッド化しやすい）
- 学びを言語化しておく（note 用）
```

### 2. テンプレート化

```
再利用テンプレートを用意:
- Twitter スレッドテンプレート
- note 裏話テンプレート
- LinkedIn 要約テンプレート
```

### 3. バッチ処理

```
月に1回「再利用デー」を設ける:
- 過去1ヶ月の記事を振り返り
- 再利用できるものをリストアップ
- まとめて再利用コンテンツを作成
```

# エンジニア向けコンテンツ戦略

技術者をターゲットにしたコンテンツの追加観点。

## エンジニア向けの特徴

| 要素 | 重視すべき点 |
|:---|:---|
| **正確性** | 技術的に正確、検証済み |
| **再現性** | コード・手順がそのまま動く |
| **深さ** | 表面的でなく原理から説明 |
| **実用性** | 実務で使える |

## 推奨プラットフォーム

| Platform | 優先度 | 理由 |
|:---|:---:|:---|
| **Zenn** | ⭐⭐⭐⭐⭐ | 技術者コミュニティ、収益化可 |
| **Qiita** | ⭐⭐⭐⭐ | 認知度高、検索流入 |
| **GitHub** | ⭐⭐⭐⭐ | コード公開、OSS 貢献 |
| **Twitter/X** | ⭐⭐⭐ | 速報、議論、ネットワーキング |
| **YouTube** | ⭐⭐⭐ | ハンズオン、長尺説明 |

## コンテンツタイプ

### 1. チュートリアル（How-to）

```markdown
# [技術] で [成果物] を構築する

## 完成形
[スクリーンショット/アーキテクチャ図]

## 前提条件
- [必要な知識]
- [必要なツール]

## 手順
### Step 1: [タイトル]
```code
[コマンド/コード]
```

### Step 2: [タイトル]
...

## トラブルシューティング
| エラー | 原因 | 解決策 |
|:---|:---|:---|

## まとめ
```

### 2. 比較記事（X vs Y）

```markdown
# [X] vs [Y]: どっちを選ぶべき？

## 結論（先に）
[ユースケース別の推奨]

## 比較表
| 観点 | X | Y |
|:---|:---|:---|
| 学習コスト | ⭐⭐⭐ | ⭐⭐ |
| パフォーマンス | ... | ... |

## 詳細比較
### [観点1]
...

## 選択基準
- X を選ぶべき場合: [条件]
- Y を選ぶべき場合: [条件]
```

### 3. トラブルシュート記事

```markdown
# [エラーメッセージ] の解決方法

## 環境
- OS: [バージョン]
- [ツール]: [バージョン]

## 症状
[エラーメッセージ全文]

## 原因
[なぜこのエラーが発生するか]

## 解決策
### 方法1: [タイトル]
```code
[コマンド]
```

## 予防策
[今後同じ問題を避けるには]
```

### 4. ベストプラクティス

```markdown
# [技術] のベストプラクティス N選

## 1. [プラクティス1]
### なぜ重要か
[理由]

### 悪い例
```code
# ❌ やりがち
```

### 良い例
```code
# ✅ 推奨
```
```

## SEO キーワード戦略

### 検索意図別

| 意図 | キーワードパターン | コンテンツタイプ |
|:---|:---|:---|
| 解決 | 「[エラー] 解決」「[問題] 方法」 | トラブルシュート |
| 学習 | 「[技術] 入門」「[技術] チュートリアル」 | チュートリアル |
| 比較 | 「[X] vs [Y]」「[X] [Y] 違い」 | 比較記事 |
| 設計 | 「[技術] ベストプラクティス」 | ベストプラクティス |

|:---|:---:|:---|
| Terraform AWS | 中 | 実務経験 + SAA |

## 記事の差別化

❌ 避けるべき:
- 公式ドキュメントのコピー
- 「やってみた」だけで学びがない
- 検証なしのコード

✅ 目指すべき:
- 実務での失敗・学びを含める
- 動作確認済みのコード
- 「なぜそうするか」の説明
- 図解・アーキテクチャ図

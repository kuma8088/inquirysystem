# Content Ideator v2.0

プロジェクト成果や学習内容を、市場価値の高いコンテンツに変換する。

## スキル連携

### 入力（前工程）

```
portfolio-ideator でプロジェクト完成
    ↓
「このプロジェクトをコンテンツ化して」
    ↓
content-ideator がプロジェクト情報を引き継ぎ

business-roadmap-advisor の発信計画
    ↓
「発信計画を詳細化して」
    ↓
content-ideator がカレンダーに落とし込み
```

### 出力（後工程）

```
コンテンツアイデア確定
    ↓
「このアイデアを記事化して」→ technical-blog-writer
「アーキテクチャ図を作成して」→ creating-drawio
```

## Step 1: ターゲット市場の特定

コンテンツ作成時、まずターゲットを特定し該当 cookbook を読み込む：

| ターゲット | 条件 | 参照ファイル |
|:---|:---|:---|
| エンジニア | 技術的な内容、コード中心 | `cookbook/engineer.md` |
| ビジネス層 | 非エンジニア、意思決定者 | `cookbook/business.md` |
| 初心者 | 入門者、キャリアチェンジ希望者 | `cookbook/beginner.md` |

コンテンツ再利用時は `cookbook/repurpose.md` を参照。

## 動作モード

### Mode 1: Trend Research（トレンド調査）

**トリガー:** 「[トピック]で何がバズってる？」「トレンド調査して」

**調査プロセス（Web検索必須）:**

1. プラットフォーム別トレンド分析
   - YouTube: 「[topic] tutorial 2024」
   - Zenn/Qiita: カテゴリ別トレンド記事
   - Twitter/X: ディスカッション・ペインポイント

2. キーワードリサーチ
   - ギャップ発見: 質問は多いが回答が少ない領域
   - ブルーオーシャン: 高需要・低競合

**出力:**
```
## 📊 トレンドリサーチ: [トピック]

### 現在のトレンド
- [トレンド1]: [根拠URL]
- [トレンド2]: [根拠URL]

### キーワード機会
| キーワード | 検索ボリューム | 競合度 | おすすめ度 |
|:---|:---:|:---:|:---:|
| [KW1] | 高 | 低 | ⭐⭐⭐⭐⭐ |
| [KW2] | 中 | 中 | ⭐⭐⭐ |

### Naoya のスキルとの相性
| キーワード | 活かせるスキル | コンテンツ案 |
|:---|:---|:---|
| [KW1] | AWS SAA | [具体案] |
| [KW2] | Terraform | [具体案] |

### 参考クリエイター
- [名前]: [URL] - [特徴・学べること]
```

### Mode 2: Project-to-Content（プロジェクト → コンテンツ化）

**トリガー:** 「このプロジェクトをコンテンツ化して」

**プロセス:**
1. プロジェクト分析（何を作った、何を学んだ、何が難しかった）
2. 複数の切り口を抽出
3. プラットフォーム別アダプテーション
4. 再利用戦略（1プロジェクト → 複数コンテンツ）

**切り口パターン:**
| パターン | 例 |
|:---|:---|
| Tutorial | 「Terraform で X を構築する方法」 |
| Lessons | 「X をデプロイして失敗した3つのこと」 |
| Comparison | 「X vs Y：どっちを選ぶべき？」 |
| Behind-the-scenes | 「なぜこのアーキテクチャを選んだか」 |
| Beginner-friendly | 「初心者でもわかる X 入門」 |

**出力:**
```
## 💡 Content Ideas from Project

### プロジェクト概要
[portfolio-ideator から引き継いだ情報]

### コンテンツ展開案（1 → 多）

| # | 切り口 | タイトル案 | Platform | 工数 |
|:---:|:---|:---|:---|:---:|
| 1 | Tutorial | [タイトル] | Zenn | 4h |
| 2 | Thread | [タイトル] | Twitter | 1h |
| 3 | Video | [タイトル] | YouTube | 8h |
| 4 | Story | [タイトル] | note | 2h |

### メインコンテンツ詳細

#### タイトル候補（A/Bテスト用）
- A: [SEO最適化タイトル]
- B: [好奇心を刺激するタイトル]
- C: [課題解決型タイトル]

#### ターゲット
- Primary: [誰に向けて]
- 解決する課題: [ペインポイント]

#### キーワード & ハッシュタグ
- SEO: [kw1], [kw2], [kw3]
- Twitter: #tag1 #tag2 #tag3

#### コンテンツ構成
1. Hook: [注目を引く導入]
2. Problem: [課題提起]
3. Solution: [メインコンテンツ]
4. CTA: [行動喚起]

### 🔗 連携スキル
- **technical-blog-writer**: 「このアイデアを記事化して」
- **creating-drawio**: 「アーキテクチャ図を作成して」
```

### Mode 3: Content Calendar（コンテンツカレンダー）

**トリガー:** 「今月のコンテンツ計画を」

**プロセス:**
1. business-roadmap-advisor の計画を参照（あれば）
2. ビジネス目標との整合
3. プラットフォーム別投稿頻度提案
4. コンテンツタイプのバランス

**出力:**
```

### 今月のテーマ
[ビジネス目標と連動したテーマ]

### 投稿計画
| Week | Platform | Type | Topic | 工数 | Status |

| W3 | note | Story | [トピック] | 2h | ⬜ |

### 月間工数
- 合計: Xh
- 週平均: Xh

| Platform | 頻度 | 理由 |
|:---|:---|:---|
```

## プラットフォーム別ガイドライン

| Platform | コンテンツタイプ | 長さ | 成功要因 |
|:---|:---|:---|:---|
| YouTube | Tutorial, Walkthrough | 10-20分 | サムネイル、最初30秒 |
| Zenn/Qiita | Deep-dive, How-to | 2000-5000字 | コードスニペット、タグ |
| note | Story, Opinion | 1500-3000字 | 個人的視点、共感性 |
| Twitter/X | Tips, Thread | 280字/10ツイート | 1ツイート目のフック |

---

## コミュニケーションスタイル

- コンテンツストラテジスト + SEO スペシャリストとして考える
- 提案は検索データで裏付け
- 同じトピックでも複数の切り口を提示
- ユーザーの時間制約を考慮
- 差別化を推進:「皆が X を投稿している、あなたは Y を投稿すべき」
- 1プロジェクト → 複数コンテンツの再利用を常に提案

# 比較記事の書き方

X vs Y を比較する記事の追加ガイドライン。

## 比較記事の特徴

| 要素 | 重視すべき点 |
|:---|:---|
| **公平性** | 両方の長所短所を示す |
| **判断基準** | 読者が選べるように |
| **実体験** | 実際に使った上での比較 |
| **結論先出し** | 忙しい読者のために |

## 構成テンプレート

```markdown
# [X] vs [Y]: どっちを選ぶべき？

## 結論（先に）

| ユースケース | 推奨 |
|:---|:---|
| [ケース1] | X |
| [ケース2] | Y |
| [ケース3] | どちらでも |

**一言で:**
- X を選ぶべき人: [条件]
- Y を選ぶべき人: [条件]

## 比較表

| 観点 | X | Y |
|:---|:---:|:---:|
| 学習コスト | ⭐⭐⭐ | ⭐⭐ |
| パフォーマンス | ⭐⭐ | ⭐⭐⭐ |
| エコシステム | ⭐⭐⭐ | ⭐⭐ |
| ドキュメント | ⭐⭐⭐ | ⭐⭐⭐ |
| 将来性 | ⭐⭐⭐ | ⭐⭐ |

## 詳細比較

### 1. [観点1]

#### X の場合
[説明]

#### Y の場合
[説明]

#### 判定
[どちらが優れているか、条件付き]

### 2. [観点2]
...

## 実際に使ってみて

### X を使った感想
[実体験]

### Y を使った感想
[実体験]

## 選択基準チェックリスト

以下に当てはまる数が多い方を選びましょう:

### X を選ぶべき場合
- [ ] [条件1]
- [ ] [条件2]
- [ ] [条件3]

### Y を選ぶべき場合
- [ ] [条件1]
- [ ] [条件2]
- [ ] [条件3]

## まとめ

[最終的な推奨と注意点]
```

## 比較記事の罠

### ❌ 避けるべきパターン

```markdown
# X vs Y

## X とは
[X の説明 1000字]

## Y とは
[Y の説明 1000字]

## まとめ
どちらも良いので好みで選びましょう。
```

**問題点:**
- 結論が曖昧
- 比較になっていない
- 読者が判断できない

### ✅ 推奨パターン

```markdown
# X vs Y

## 結論: [ケース別の推奨を先に]

## 比較表: [一目でわかる]

## 詳細: [観点ごとに対比]

## 判断基準: [読者が自分で選べる]
```

## 公平な比較のコツ

### 1. 両方を実際に使う

```markdown
✅ 「実際に X で構築し、同じものを Y で作り直して比較」
❌ 「ドキュメントを読んで比較」
```

### 2. 同じ観点で比較

```markdown
✅ 
| 観点 | X | Y |
|:---|:---|:---|
| セットアップ時間 | 10分 | 30分 |

❌
| X | Y |
|:---|:---|
| セットアップが簡単 | 機能が豊富 |
```

### 3. 条件付きで判定

```markdown
✅ 「小規模プロジェクトなら X、大規模なら Y」
❌ 「X の方が良い」
```

## 比較観点の例

### 技術選定の場合

| 観点 | 説明 |
|:---|:---|
| 学習コスト | 習得までの時間 |
| パフォーマンス | 速度、リソース効率 |
| スケーラビリティ | 規模拡大への対応 |
| エコシステム | ライブラリ、ツール |
| コミュニティ | 情報量、サポート |
| ドキュメント | 公式ドキュメントの質 |
| 将来性 | 開発の活発さ、採用企業 |
| コスト | ライセンス、運用コスト |

### ツール比較の場合

| 観点 | 説明 |
|:---|:---|
| 価格 | 無料/有料、料金体系 |
| 機能 | 必要な機能の有無 |
| UI/UX | 使いやすさ |
| 連携 | 他ツールとの統合 |
| サポート | カスタマーサポート |

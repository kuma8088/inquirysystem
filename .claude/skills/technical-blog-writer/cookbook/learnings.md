# 学び・体験記事の書き方

失敗談、気づき、成長を共有する記事の追加ガイドライン。

## 学び記事の特徴

| 要素 | 重視すべき点 |
|:---|:---|
| **共感性** | 読者が「わかる」と思える |
| **具体性** | 抽象論ではなく実例 |
| **再現性** | 読者が活かせる学び |
| **正直さ** | 失敗を隠さない |

## 構成テンプレート

### パターン1: 失敗談

```markdown
# [失敗したこと] から学んだ N つのこと

## 何が起きたか

[状況説明: いつ、どんなプロジェクトで]

## 失敗の詳細

### やったこと
[具体的な行動]

### 結果
[何が起きたか]

### 原因
[なぜそうなったか]

## 学び

### 1. [学び1]
[詳細と今後どうするか]

### 2. [学び2]
[詳細と今後どうするか]

## 今ならこうする

[同じ状況で今ならどう対処するか]

## 同じ失敗をしないために

[読者へのアドバイス]
```

### パターン2: 成長記録

```markdown
# [スキル/技術] を習得するまでの道のり

## Before
- 状態: [どんな状態だったか]
- 課題: [何ができなかったか]
- きっかけ: [なぜ学ぼうと思ったか]

## 学習プロセス

### Phase 1: [期間]
[やったこと、つまずき]

### Phase 2: [期間]
[やったこと、突破口]

### Phase 3: [期間]
[やったこと、定着]

## After
- 状態: [今どうなったか]
- できるようになったこと
- まだ苦手なこと

## 振り返って

### やって良かったこと
- [具体的な行動]

### やらなくて良かったこと
- [無駄だったこと]

### もっと早くやれば良かったこと
- [後から気づいたこと]

## これから学ぶ人へ

[アドバイス]
```

### パターン3: 気づき共有

```markdown
# [技術/仕事] で気づいた意外なこと

## 背景

[どんな状況でこの気づきを得たか]

## 気づき

### Before: こう思っていた
[以前の認識]

### After: 実はこうだった
[新しい認識]

## なぜそう言えるか

[根拠、実例]

## この気づきをどう活かすか

[今後の行動変化]
```

## 学び記事の強み

技術記事との差別化:

| 技術記事 | 学び記事 |
|:---|:---|
| 正解を教える | 失敗を共有する |
| 再現性重視 | 共感性重視 |
| 誰でも書ける | 体験者だけが書ける |
| 検索流入 | SNS シェア |

## 正直に書くコツ

### ❌ 避けるべき

```markdown
少し苦労しましたが、最終的には成功しました。
```

### ✅ 推奨

```markdown
3日間ハマり続けて、何度も「もう無理」と思いました。
解決したきっかけは、諦めて寝ようとした瞬間に
「あれ、もしかして...」と気づいたことでした。
```

## 共感を生む表現

### 読者の気持ちを代弁

```markdown
✅ 「ドキュメントを読んでも意味不明、という経験はありませんか？」
✅ 「なんでこんな簡単なことに3時間もかかるんだ、と自己嫌悪に陥りました」
```

### 具体的な感情

```markdown
✅ 「エラーが消えた瞬間、思わず声が出ました」
✅ 「本番で障害が起きた時、手が震えました」
❌ 「嬉しかったです」
❌ 「大変でした」
```

## 学びの言語化

### STAR フレームワーク

```
Situation（状況）: どんな状況で
Task（課題）: 何をしようとして
Action（行動）: 何をして
Result（結果）: どうなったか
```

### 学びの深掘り

```
1. 何が起きたか（事実）
2. なぜそうなったか（原因）
3. 何を学んだか（学び）
4. 今後どうするか（行動変化）
```

## note での書き方

```markdown
# note 向けの特徴

- より個人的な視点
- 感情を含める
- 「私」を主語に
- 技術詳細より学びを前面に
- 画像・図を多めに
```
